당신은 프로덕션 경험이 풍부한 시니어 풀스택 엔지니어입니다.
우리는 급식E를 Next.js(App Router)+TS+Tailwind+Prisma+Postgres로 개발 중이며,
1단계에서 세션 쿠키 기반 Auth와 /dashboard 보호를 만들었고,
2단계에서 School 매핑 + NEIS 급식 + 피드백→포인트 + 랭킹까지 구현해 둔 상태입니다.

이제 “구현 3단계”를 설계하고 실제 코드를 수정/추가해 완성하세요.
3단계 목표는: (1) 클린플레이트 사진 업로드, (2) (Mock) AI 판정/상태머신, (3) 승인 시 포인트 지급, (4) 관리자 리뷰 화면/권한 입니다.
UI 톤은 Apple처럼 미니멀: 흰 배경, 얇은 구분선, 절제된 블루 포인트, 여백/타이포 중심. 과한 그림자/애니메이션 금지.

=====================================================
1) 저장소(이미지 업로드) 아키텍처: S3 호환 + 로컬 MinIO
=====================================================
- 프로덕션 대비를 위해 S3 호환 API를 사용.
- 로컬 개발은 docker-compose에 MinIO 서비스를 추가해서 버킷을 사용.
- Next.js 서버에서만 S3 자격증명을 사용(클라이언트 노출 금지).

필수 env:
- S3_ENDPOINT (예: http://localhost:9000)
- S3_REGION (예: us-east-1)
- S3_ACCESS_KEY_ID
- S3_SECRET_ACCESS_KEY
- S3_BUCKET (예: gupsik-e)
- S3_PUBLIC_URL (예: http://localhost:9000/gupsik-e)  // 개발용 단순 공개 URL

서버 유틸:
- src/server/storage.ts
  - uploadImage(fileBuffer, contentType, key) -> { url, key }
  - (선택) generateKey(userId, date, originalName)

업로드 제한:
- 이미지 MIME만 허용: image/jpeg, image/png, image/webp
- 최대 5MB
- EXIF/메타데이터 처리는 다음 단계로 미룸(주석으로 확장 포인트만)

3) (Mock) AI 판정 로직: 추후 실제 모델로 교체 가능한 구조
=====================================================
- src/server/cleanplate-ai.ts
  - analyzeCleanPlate(imageUrl: string) -> { score: number, verdict: "APPROVE"|"REVIEW"|"REJECT", details: any }
- 지금은 Mock:
  - score는 pseudo-random 또는 간단 규칙
  - verdict:
    - score >= 0.85 => APPROVE(자동승인)
    - 0.5~0.85 => REVIEW(관리자 검토)
    - < 0.5 => REJECT(자동거절 또는 관리자 검토; 이번 단계는 REJECT로 처리)

중요:
- “AI 자동 승인”이 나가도 포인트는 1회만 지급(트랜잭션 + pointsAwarded 체크)

4) API 라우트(백엔드) 추가
=====================================================
A) POST /api/cleanplate/upload
- multipart/form-data:
  - file: 이미지 파일(필수)
  - date: YYYY-MM-DD (옵션, 없으면 서버에서 오늘로)
- 세션 필수
- 처리:
  1) 오늘(또는 date) 기준으로 (userId, date) 중복이면 에러
  2) S3(MinIO)에 업로드 -> imageUrl, imageKey 확보
  3) CleanPlateSubmission 생성 status=PENDING
  4) 바로 Mock AI 분석 실행:
     - verdict=APPROVE면 status=AUTO_APPROVED로 변경 + 포인트 지급(+100)까지 한 번에 처리
     - verdict=REVIEW면 status=PENDING 유지
     - verdict=REJECT면 status=REJECTED로 변경(포인트 없음)
- 응답:
  { ok:true, submission:{ id, status, imageUrl, date }, pointsDelta, newPoints }

B) GET /api/cleanplate/today
- 오늘 기준 내 제출 상태 반환(없으면 null)

C) GET /api/cleanplate/history?days=7
- 최근 N일 제출 목록(상태/날짜/포인트)

D) (ADMIN) GET /api/admin/cleanplate?status=PENDING
- PENDING 목록(학교 단위로만 보여주기: 같은 schoolId)
- 개인정보 최소화: “13번” 형태로만 표시할 수 있도록 user의 studentNumber만 제공(실명 금지)

E) (ADMIN) POST /api/admin/cleanplate/{id}/review
- body: { action: "APPROVE"|"REJECT", note?: string }
- 승인 시:
  - status=APPROVED, reviewedAt, reviewedByUserId, reviewNote 저장
  - pointsAwarded==0이면 +100 지급(ledger 기록 + user.points 증가) 트랜잭션
- 거절 시:
  - status=REJECTED, reviewedAt 등만 저장(포인트 없음)

에러 응답 형식 통일:
- { ok:false, error:{ code, message } }
